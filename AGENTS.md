# AGENTS

## Правила работы с vault

- Всю рабочую документацию по проекту веди в хранилище vault.
- При любых изменениях в коде, конфигурации, тестах, CI/CD или структуре проекта обязательно обновляй соответствующую документацию в vault в рамках той же задачи.
- Не оставляй изменения без документирования: описание должно отражать текущее поведение системы.
- Если изменение временное или экспериментальное, это также должно быть явно отмечено в документации vault.
